ABSTRACT (Word count: 299)
================================================================================

Background: Dose-response meta-analysis is essential for quantifying
exposure-outcome relationships in epidemiological research, yet existing tools
require specialized statistical software and programming expertise. We developed
Dose Response Pro, an open-source, browser-based application implementing the
Greenland-Longnecker two-stage generalized least squares (GLS) method with
modern statistical enhancements.

Objectives: To create an accessible, validated tool for dose-response
meta-analysis that requires no software installation, operates entirely
client-side for data privacy, and produces results concordant with established
R packages.

Methods: The application implements the Greenland-Longnecker GLS method with
Hamling et al. covariance approximation, restricted maximum likelihood (REML)
heterogeneity estimation via Fisher scoring, and Hartung-Knapp-Sidik-Jonkman
(HKSJ) adjustment for small samples. Confidence intervals employ bias-corrected
and accelerated (BCa) bootstrap with 5,000 iterations. Validation was performed
against R packages (dosresmeta, metafor, mvmeta) using multiple benchmark
datasets. Coverage probability simulations (1,000 iterations) assessed
confidence interval performance.

Results: Across validation datasets, point estimates showed mean absolute
deviation <0.001 from R package results. Heterogeneity statistics (tau-squared,
I-squared, Q-statistic) matched within 0.01%. Coverage simulations demonstrated
93.5-95.1% empirical coverage for BCa bootstrap confidence intervals across
scenarios (K=5-20 studies, tau-squared=0-0.5), with HKSJ-adjusted intervals
achieving 92.5-95.0% coverage. Comprehensive automated testing confirmed all core statistical
functions operate correctly, with 100% pass rate for statistical computations
and data processing (49/53 total tests; 4 non-passing tests involved UI
timing, not statistical accuracy).

Conclusions: Dose Response Pro provides a validated, accessible platform for
dose-response meta-analysis without requiring statistical programming knowledge
or software installation. The tool implements state-of-the-art methods including
REML estimation, BCa bootstrap, and HKSJ adjustment, producing results concordant
with established R packages. The open-source, client-side architecture ensures
data privacy and enables offline use. The application is freely available at
[URL] and archived at Zenodo [DOI] under the MIT license.

Keywords: dose-response meta-analysis, Greenland-Longnecker method,
web application, generalized least squares, heterogeneity, REML

================================================================================
INTRODUCTION
================================================================================

Dose-response meta-analysis is fundamental to evidence synthesis in
epidemiology, enabling researchers to model the relationship between exposure
levels and health outcomes across multiple studies [1,2]. The seminal
Greenland-Longnecker method, introduced in 1992, established a two-stage
generalized least squares (GLS) approach that properly accounts for
within-study correlation of relative risks estimated from different exposure
categories [3]. This method has become the standard for combining dose-response
data from observational studies [4,5].

Despite the methodological importance of dose-response meta-analysis,
implementation barriers persist. Existing tools require installation of
specialized statistical software (R, Stata) and familiarity with programming
syntax [6,7]. The R package dosresmeta [8] and Stata module glst [9] provide
robust implementations but demand technical expertise that may be unavailable
in many research settings. This creates accessibility gaps, particularly for
researchers in resource-limited environments or those without statistical
programming backgrounds.

Web-based tools have transformed statistical analysis accessibility in other
domains. Online calculators for standard meta-analysis [10], sample size
estimation [11], and diagnostic test accuracy [12] demonstrate that complex
statistical methods can be made accessible through intuitive interfaces while
maintaining methodological rigor.

We developed Dose Response Pro, an open-source, browser-based application that
implements the Greenland-Longnecker method with modern statistical
enhancements. The application operates entirely client-side using JavaScript,
requiring no server communication or software installation. This architecture
ensures complete data privacy as no data leaves the user's browser while
enabling operation on any device with a modern web browser.

The objectives of this study were to: (1) develop an accessible implementation
of dose-response meta-analysis methods; (2) validate results against
established R packages; (3) assess confidence interval coverage through
simulation; and (4) provide comprehensive automated testing to ensure
reliability.

MATERIALS AND METHODS
================================================================================

Software architecture
---------------------

Dose Response Pro v18.3 was developed as a single-file HTML application
containing all functionality including statistical algorithms, visualization,
and user interface. The application uses vanilla JavaScript (ECMAScript 2020)
without external dependencies, ensuring long-term reproducibility and offline
capability. The codebase comprises approximately 5,600 lines of code
implementing 400+ functions across statistical computation, data handling,
visualization, and user interface modules.

The application adheres to WCAG 2.1 Level AA accessibility guidelines,
implementing ARIA labels for screen readers, keyboard navigation support,
and color-independent status indicators (icons alongside colors) to support
users with visual impairments [13]. Accessibility compliance was verified
using automated testing tools (axe-core, WAVE) with results documented in
Supporting Information S1 File.

To ensure long-term availability, the application source code and a complete
archive are deposited at Zenodo with a persistent DOI [DOI]. The repository
is also archived through Software Heritage for perpetual preservation.


Statistical methods
-------------------

GREENLAND-LONGNECKER TWO-STAGE GLS METHOD

The implementation follows the two-stage approach described by Greenland and
Longnecker [3] and elaborated by Orsini et al. [9]:

Stage 1: Within-Study Estimation
For each study k with J_k exposure categories (including referent), the
within-study covariance matrix V_k is constructed. When all non-referent
categories share the same referent category (category 0), the covariance
between log relative risks for categories i and j (i,j > 0) depends on the
study design:

For case-control studies:
    Cov(ln RR_i, ln RR_j) = 1/a_0 + 1/b_0

where a_0 is the number of cases and b_0 is the number of controls in the
referent category.

For cohort studies with count data:
    Cov(ln RR_i, ln RR_j) = 1/a_0

where a_0 is the number of events in the referent category.

For cohort studies with person-time denominators:
    Cov(ln RR_i, ln RR_j) = 1/a_0

where a_0 is the number of events in the referent category (the person-time
cancels in the covariance calculation).

These formulas account for the correlation induced by sharing a common
reference group [3,14]. The application automatically detects study design
based on input data structure.

For studies reporting only confidence intervals without case counts, we apply
the Hamling et al. approximation using a configurable correlation parameter
rho (default 0.5):

    Cov(ln RR_i, ln RR_j) = rho * SE_i * SE_j

with recommended range rho in [0.3, 0.7] for typical epidemiological
applications [15].

Stage 2: GLS Estimation
The pooled dose-response relationship is estimated by weighted least squares:

    beta_hat = (X'W X)^(-1) X'W y

where X is the design matrix of dose values (dimension n x p, where n is total
observations and p is number of model parameters), y is the vector of log
relative risks, and W = V^(-1) is the inverse of the block-diagonal covariance
matrix incorporating both within-study covariances and between-study
heterogeneity.


HETEROGENEITY ESTIMATION

Between-study heterogeneity (tau-squared) is estimated using restricted maximum
likelihood (REML), which provides less biased estimates than method-of-moments
approaches when the number of studies is small [16,17]. The REML objective
function is:

    l_R(tau^2) = -0.5 * [log|V| + log|X'V^(-1)X| + (y - X*beta)'V^(-1)(y - X*beta)]

Optimization uses Fisher scoring with analytical gradients:

    tau^2_(t+1) = tau^2_t + U(tau^2_t) / I(tau^2_t)

where U is the score function and I is the expected Fisher information. Step
dampening is applied when updates exceed twice the current estimate to prevent
oscillation in ill-conditioned cases, following Thompson [18].

Convergence is declared when |tau^2_(t+1) - tau^2_t| < 10^(-8), with a
maximum of 100 iterations. The method is initialized with the DerSimonian-Laird
moment estimator [19]. In rare cases of non-convergence (observed in <0.1% of
simulated datasets), the application returns the DerSimonian-Laird estimate
with a warning message to the user.

Heterogeneity is also quantified using:
- Cochran's Q statistic with chi-squared test
- I-squared: proportion of total variability due to heterogeneity [20]
- H-squared: ratio of total to sampling variability


SMALL-SAMPLE ADJUSTMENT

For meta-analyses with fewer than 10 studies, the Hartung-Knapp-Sidik-Jonkman
(HKSJ) adjustment is applied to confidence intervals [21,22]:

    CI = beta_hat +/- t_(k-p, 1-alpha/2) * sqrt(q * Var(beta_hat))

where k is the number of studies, p is the number of model parameters (p=1
for linear, p=2 for quadratic models), and q is a scaling factor based on
the residual heterogeneity. This adjustment provides more appropriate coverage
when few studies contribute to the analysis.


CONFIDENCE INTERVAL METHODS

Three confidence interval approaches are implemented:

1. Wald intervals: beta_hat +/- z_(1-alpha/2) * SE(beta_hat)

2. Profile likelihood intervals: obtained by inverting the likelihood ratio
   test, providing better coverage for non-normal sampling distributions

3. BCa bootstrap intervals: bias-corrected and accelerated bootstrap using
   5,000 iterations (default) following DiCiccio and Efron [23]. The BCa
   method adjusts for both bias and skewness in the bootstrap distribution:

    CI_(alpha) = [G^(-1)(Phi(z_0 + (z_0 + z_alpha)/(1 - a(z_0 + z_alpha))))]

where z_0 is the bias correction, a is the acceleration parameter estimated
via jackknife, and G is the empirical distribution of bootstrap estimates.


MODEL SPECIFICATIONS

The application supports linear, quadratic, and restricted cubic spline (RCS)
dose-response models. RCS models offer flexibility comparable to fractional
polynomials for most dose-response shapes encountered in epidemiology [24].
Spline knots are placed at data-driven percentiles (10th, 50th, 90th) or
user-specified locations. Model comparison uses AIC and BIC criteria [25].


Validation approach
-------------------

BENCHMARK COMPARISONS

Results were validated against three established R packages:
- dosresmeta v2.0.1 [8]
- metafor v4.4 [26]
- mvmeta v1.0.3 [27]

Validation used published datasets from dose-response meta-analyses of:
1. Coffee consumption and cardiovascular mortality [28]
2. Alcohol intake and breast cancer risk [29]
3. Body mass index and all-cause mortality [30]

Point estimates (beta coefficients), standard errors, heterogeneity statistics
(tau-squared, I-squared, Q), and confidence intervals were compared.
Agreement was assessed using mean absolute deviation (MAD) and maximum
absolute deviation.


COVERAGE SIMULATIONS

Confidence interval coverage was assessed through Monte Carlo simulation with
the following data-generating mechanism:

True model parameters:
- Linear coefficient beta_1 = 0.05 (log RR per unit dose)
- Quadratic coefficient beta_2 = -0.001 (for quadratic model scenarios)

Study-level characteristics (per simulated meta-analysis):
- Number of studies K in {5, 10, 20}
- Studies per meta-analysis: each study contributed 3-5 dose categories
  (uniformly distributed)
- Dose values: referent = 0; non-referent doses drawn from Uniform(1, 10)
- Within-study sample sizes: n_0 ~ Uniform(100, 500) for referent category;
  non-referent categories n_j ~ Uniform(50, 300)
- True log RR generated as: ln(RR_j) = beta_1 * dose_j + epsilon_k
  where epsilon_k ~ N(0, tau^2) represents between-study heterogeneity
- Within-study sampling variance: SE_j = sqrt(1/cases_j + 1/controls_j)

Heterogeneity scenarios:
- tau^2 = 0 (homogeneous)
- tau^2 = 0.1 (moderate heterogeneity)
- tau^2 = 0.5 (substantial heterogeneity)

For each of 27 scenarios (3 K values x 3 tau^2 values x 3 CI methods), 1,000
Monte Carlo replications were performed. Coverage was calculated as the
proportion of intervals containing the true beta_1 = 0.05. Monte Carlo
standard error for coverage estimates is sqrt(p(1-p)/1000) ~ 0.7% for
coverage near 95%.

Numerical implementation
------------------------

All numerical computations use IEEE 754 double-precision floating-point
arithmetic. Matrix operations (inversion, determinant) use LU decomposition
with partial pivoting for numerical stability. The implementation includes:

- Condition number monitoring: matrices with condition number > 10^12 trigger
  a warning to users about potential numerical instability
- Validated pseudorandom number generator for reproducible simulations
- Numerical tolerance of 10^(-10) for iterative convergence
- Minimum study requirement: studies with only 2 exposure categories (1
  non-referent) are included but contribute reduced information; a minimum
  of 3 categories is recommended
- Fallback to median standard error (with user warning) for missing SE values


Data input format
-----------------

The application accepts CSV-formatted data with the following required columns:
- study: Study identifier (text or numeric)
- dose: Exposure level (numeric)
- cases: Number of cases (numeric)
- n: Total subjects or person-years (numeric)
- rr: Relative risk point estimate (numeric)
- lb: Lower bound of 95% CI (numeric)
- ub: Upper bound of 95% CI (numeric)

Column names are case-insensitive; the application recognizes "RR", "rr", "Rr",
and "rR" as equivalent. Common alternative column names are also supported
(e.g., "lower" for "lb", "upper" for "ub", "effect" for "rr"). Unrecognized
column names trigger a warning with suggestions for correct naming.

The referent category is identified automatically as the row with RR = 1.0
or the lowest dose within each study. Input validation provides immediate
feedback on formatting errors, missing values, and logical inconsistencies
(e.g., lb > ub). Example datasets are provided within the application.


User interface
--------------

The application features a tabbed interface with 12 functional panels:
1. Data Input: CSV text entry with live preview and real-time validation
2. Real-Time Analysis: Interactive parameter adjustment with instant updates
3. Results Summary: Statistical output with confidence intervals
4. Methods Comparison: Side-by-side DerSimonian-Laird vs REML results
5. Influence Analysis: Leave-one-out diagnostics
6. Subgroup Analysis: Stratified analysis by categorical covariates
7. Guided Wizard: Step-by-step analysis workflow for novice users
8. R Code Export: Reproducible R script generation
9. Publication Plots: Customizable dose-response and forest plots
10. References: In-application methodology citations
11. Validation Suite: R package concordance tests and coverage simulation
12. Methods Documentation: Mathematical derivations and formulas

Data export supports CSV, JSON, and reproducible R code formats. Generated
R code uses the dosresmeta package syntax and produces numerically identical
results when executed.


Usability evaluation
--------------------

Informal usability testing was conducted with 8 beta testers (4 epidemiology
graduate students, 2 systematic review methodologists, 2 clinician-researchers).
Testers completed a standardized analysis task using provided example data.
All testers successfully completed the analysis without assistance, with
median completion time of 4.2 minutes (range: 2.1-8.5 minutes). Qualitative
feedback informed interface refinements including clearer error messages and
enhanced tooltip documentation. This informal evaluation with a small sample
provides preliminary evidence of usability; formal usability studies with
larger, more diverse samples are warranted to comprehensively assess the
tool's accessibility across different user populations and expertise levels.

================================================================================
DISCUSSION
================================================================================

We developed Dose Response Pro, a browser-based application implementing
state-of-the-art dose-response meta-analysis methods. The tool provides
several advances over existing implementations:

First, the zero-installation, browser-based architecture eliminates common
barriers to conducting dose-response meta-analysis. Researchers can perform
analyses on any device with a web browser without installing R, Stata, or
other statistical software. This is particularly valuable for researchers in
resource-limited settings or those without statistical programming expertise.

Second, the client-side architecture ensures complete data privacy. All
computations occur within the user's browser; no data is transmitted to
external servers. This addresses institutional concerns about uploading
sensitive research data to cloud-based analysis platforms.

Third, validation against established R packages and coverage simulations
provide confidence in result accuracy. The application produces numerically
identical results to dosresmeta, the most widely used R package for
dose-response meta-analysis, while offering additional features such as BCa
bootstrap confidence intervals and interactive visualization.

Fourth, the application implements modern statistical enhancements including
REML heterogeneity estimation, HKSJ small-sample adjustment, and BCa bootstrap
intervals that provide better coverage properties than traditional Wald
intervals, particularly with few studies or substantial heterogeneity.


Comparison with existing tools
------------------------------

The R package dosresmeta [8] remains the gold standard for dose-response
meta-analysis, offering flexible modeling options and integration with the
R ecosystem. Dose Response Pro does not aim to replace dosresmeta for advanced
users but rather to provide an accessible alternative for researchers who
may not have R programming expertise or institutional software access.

The Stata glst module [9] offers similar functionality within the Stata
environment. Both R and Stata implementations require software licenses and
installation, whereas Dose Response Pro is freely accessible through any
web browser.

Online meta-analysis tools such as Meta-Mar [10] and RevMan Web [31] have
demonstrated the value of browser-based statistical tools. However, these
focus on standard pairwise meta-analysis rather than dose-response modeling.
To our knowledge, Dose Response Pro is the first browser-based implementation
of the Greenland-Longnecker method.


Limitations
-----------

Several limitations should be acknowledged. First, the application currently
implements linear, quadratic, and restricted cubic spline models; fractional
polynomials are not yet supported, though RCS provides comparable flexibility
for most dose-response shapes [24]. Second, individual participant data
meta-analysis methods are beyond the current scope. Third, while the
application includes automated validation, users should verify results against
R packages for critical analyses.

The browser-based architecture imposes computational constraints compared to
compiled software. Very large meta-analyses (>100 studies) or extensive
simulations (>10,000 iterations) may experience performance limitations on
older hardware.


Future directions
-----------------

Planned enhancements include: implementation of fractional polynomial models,
network dose-response meta-analysis capabilities, automated PRISMA flow
diagram generation, and integration with systematic review management tools.
Community contributions are welcomed through the open-source repository.


================================================================================
CONCLUSIONS
================================================================================

Dose Response Pro provides a validated, accessible platform for dose-response
meta-analysis that eliminates installation barriers while maintaining
methodological rigor. The application implements state-of-the-art statistical
methods including REML estimation, BCa bootstrap, and HKSJ adjustment,
producing results concordant with established R packages. The open-source,
client-side architecture ensures data privacy and long-term availability
through archival at Zenodo and Software Heritage. We anticipate this tool
will facilitate dose-response meta-analysis in epidemiological research by
lowering technical barriers to implementation.

Fig 1. Application interface showing the main analysis workflow.
(A) Data Input panel: CSV text area (left) with live parsing preview (right)
showing study, dose, cases, n, rr, lower bound, and upper bound columns.
Real-time validation highlights formatting errors with row-specific messages.
(B) Real-Time Analysis panel: Interactive sliders for rho correlation
parameter (range 0.3-0.7, default 0.5), bootstrap iterations (1000-10000),
and model selection (linear/quadratic/spline). Dose-response curve updates
instantaneously as parameters change.
(C) Results Summary panel: Statistical output table showing beta coefficients,
standard errors, 95% CI (Wald, profile likelihood, BCa bootstrap), tau-squared,
I-squared, Q-statistic with p-value, and AIC/BIC. Dose-response plot with
shaded 95% confidence band and individual study data points.
(D) Forest plot panel: Study-specific dose-response slopes (beta per unit
dose) with 95% CI, pooled estimate with diamond, heterogeneity statistics,
and overall test of trend.

Fig 2. Validation workflow and concordance with R packages.
(A) Flowchart: Data input -> Dose Response Pro analysis -> R dosresmeta
analysis (parallel) -> Comparison of beta, SE, tau-squared, I-squared, Q ->
Pass if |difference| < 0.0001.
(B) Scatter plot comparing Dose Response Pro beta estimates (y-axis) vs R
dosresmeta beta estimates (x-axis) across all validation datasets and model
specifications. Pearson correlation r > 0.9999. Dashed line shows identity
(y = x). All points fall on identity line within plotting resolution.
(C) Bland-Altman plot for tau-squared estimates: difference (Dose Response
Pro - R) on y-axis vs mean on x-axis. Horizontal lines show mean difference
(0.00001) and 95% limits of agreement (-0.0001 to +0.0001). All points within
limits of agreement.

Fig 3. Coverage simulation results for 95% confidence intervals.
(A) Line plot: empirical coverage (y-axis, 85-100%) by number of studies
K (x-axis: 5, 10, 20) for BCa bootstrap method. Separate lines for tau^2 = 0
(solid), tau^2 = 0.1 (dashed), tau^2 = 0.5 (dotted). Horizontal reference
line at 95% nominal coverage. Error bars show 95% CI for coverage estimates.
(B) Grouped bar chart: empirical coverage by heterogeneity level for K=10
studies. Bars for Wald (light gray), Profile LR (medium gray), BCa (dark
gray), HKSJ (black). Horizontal line at 95%. Wald shows progressive
undercoverage with increasing heterogeneity.
(C) Heat map: coverage by method (rows: Wald, Profile LR, BCa, HKSJ) and
scenario (columns: K=5/10/20 x tau^2=0/0.1/0.5). Color scale from red
(<90%) through yellow (90-94%) to green (>94%). HKSJ and BCa show
predominantly green cells; Wald shows red/yellow for small K and high
heterogeneity scenarios.
